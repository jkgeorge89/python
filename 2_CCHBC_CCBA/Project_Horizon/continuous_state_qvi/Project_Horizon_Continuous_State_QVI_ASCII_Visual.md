# Project Horizon - Continuous State QVI ASCII Visual

Generated by `generate_qvi_ascii_visual.py`.

## Purpose

This note is a simplified visual aid for explaining the continuous-state QVI workstream.

The key distinction is:

- a true **tree** has one node at `t0` and fans outward as time moves to the right; and
- the QVI solver is better visualized as a **time-by-price grid** with a strategic decision problem at each grid point.

---

## 1. True Tree Shape

If you want a literal tree picture, this is the right shape.

```text
time -------------------------------------------------------------->

Read by column:
- t0 has 1 node
- t1 has 2 vertical nodes
- t2 has 3 vertical nodes
- t3 has 4 vertical nodes

+---------+----+----+----+----+
| state   | t0 | t1 | t2 | t3 |
+---------+----+----+----+----+
| state 3 |    |    |    | o  |
| state 2 |    |    | o  | o  |
| state 1 |    | o  | o  | o  |
| state 0 | o  | o  | o  | o  |
+---------+----+----+----+----+
```

That is the correct mental picture for a branching lattice.

---

## 2. QVI Time-By-Price Grid

The QVI solver is not a single fanning tree. Time is horizontal and price is vertical.

```text
+-------+------+------+------+------+------+------+
| price | day0 | day1 | day2 | day3 | day4 | day5 |
+-------+------+------+------+------+------+------+
| 112   | o    | o    | o    | o    | o    | o    |
| 109   | o    | o    | o    | o    | o    | o    |
| 106   | o    | o    | o    | o    | o    | o    |
| 103   | o    | o    | o    | o    | o    | o    |
| 100   | o    | o    | o    | o    | o    | o    |
| 97    | o    | o    | o    | o    | o    | o    |
+-------+------+------+------+------+------+------+
```

Each cell is one state point on the grid at which the solver can evaluate value.

---

## 3. One-Step Transition Intuition

For one current grid point, the next-day continuation value comes from a distribution of nearby next-day price states.

```text
+-------+-------+----------------------+
| price | day t | day t+1 neighborhood |
+-------+-------+----------------------+
| 112   |       | o                    |
| 109   |       | o o o                |
| 106   | X     | o o o o o            |
| 103   |       | o o o                |
| 100   |       | o                    |
+-------+-------+----------------------+
```

`X` is the current point whose continuation value is being computed.

---

## 4. What Lives At Each Grid Point

Each point is not just price and time. It is:

```text
Value(day t, stock price S, remaining Titan stake q)
```

So one grid point really means:

```text
(day, stock price, remaining Titan stake)
```

---

## 5. Fine Partial-Exercise Ladder

From `q = 25.00%`, Olympus compares a fine menu of admissible call sizes rather than a few coarse jumps.

```text
+--------------------------------+----------------------+
| Olympus action from q = 25.00% | Residual Titan stake |
+--------------------------------+----------------------+
| wait                           | 25.00%               |
| call 5.00%                     | 20.00%               |
| call 5.25%                     | 19.75%               |
| call 5.50%                     | 19.50%               |
| call 5.75%                     | 19.25%               |
| ...                            | ...                  |
| call 17.75%                    | 7.25%                |
| call 18.00%                    | 7.00%                |
| call 25.00%                    | 0.00%                |
+--------------------------------+----------------------+
```

So the 5% minimum tranche is still respected, but the residual stake is represented on a much finer ladder.

---

## 6. Overlap Node Logic

At each relevant point, the solver compares:

```text
WAIT
CALL some admissible amount
PUT the residual stake if Titan's right is live
```

In the overlap period, the value is the result of a strategic max/min step at that grid point.

---

## 7. Practical Reading Guide

If you are explaining this in a meeting, the cleanest sequence is:

1. Show the tree first so people see the usual branching intuition.
2. Then say: the QVI method is not that picture; it is a dense grid over time and price.
3. Then show the action ladder to explain that partial exercise is solved on a fine 0.25% control grid.

That is the cleanest honest ASCII representation of what the QVI solver is doing.
